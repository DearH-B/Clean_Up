# 실제 집 제품 등록 테스트 체크리스트

작성 기준: 2026-06-15

## 테스트 기록 방법

각 항목 옆에 다음 중 하나를 적는다.

- `통과`: 기대한 대로 동작
- `불편`: 동작하지만 사용하기 어려움
- `실패`: 오류, 멈춤, 데이터 손실 또는 잘못된 결과
- `해당 없음`: 현재 집이나 기기에서 확인할 수 없음

문제가 생기면 아래 정보도 남긴다.

- 공간과 제품 종류
- 브랜드와 모델명
- 수행한 순서
- 기대한 결과와 실제 결과
- 화면 캡처
- 앱을 다시 실행해도 문제가 유지되는지

## 에뮬레이터에서 확인할 체크리스트

### 1. 공간 구성

- [ ] 실제 집의 모든 공간을 만들 수 있다.
- [ ] 방1, 방2처럼 비슷한 공간 이름을 구분하기 쉽다.
- [ ] 같은 이름의 공간을 실수로 중복 생성하지 않는다.
- [ ] 공간 설명이 제품 구성을 이해하는 데 도움이 된다.
- [ ] 공간을 추가한 직후 해당 공간의 제품 등록으로 자연스럽게 이어진다.
- [ ] 빈 공간, 제품이 많은 공간 모두 화면이 어색하지 않다.
- [ ] 공간 삭제 경고가 삭제 범위를 충분히 설명한다.
- [ ] 공간 삭제 후 다른 공간의 제품은 유지된다.

### 2. 모든 가전·가구 등록

- [ ] 정확한 모델명을 아는 제품을 검색해 등록한다.
- [ ] 브랜드만 아는 제품을 등록한다.
- [ ] 모델명을 모르는 제품을 일반 제품군으로 등록한다.
- [ ] 카탈로그에 없는 가전제품을 직접 등록한다.
- [ ] 소파, 침대, 책상, 의자, 수납장 같은 가구를 등록한다.
- [ ] 같은 모델을 두 대 등록하고 별칭으로 구분한다.
- [ ] 같은 제품을 같은 공간에 다시 등록할 때 중복 안내가 나온다.
- [ ] 같은 모델을 다른 공간에 등록할 수 있다.
- [ ] 별칭, 구매일, 설치일과 메모가 저장된다.
- [ ] 제품 등록 도중 이전 단계로 돌아가도 입력값이 유지된다.
- [ ] 등록을 취소한 제품이 목록에 남지 않는다.
- [ ] 제품 등록 직후 관리법 보기와 나중에 보기가 모두 정상 동작한다.

### 3. 검색 품질

- [ ] 모델명 전체로 검색하면 정확한 제품이 첫 번째에 나온다.
- [ ] 모델명의 하이픈과 공백을 다르게 입력해도 검색된다.
- [ ] 영문 대소문자가 달라도 검색된다.
- [ ] 모델명 한두 글자를 틀려도 가까운 후보가 나온다.
- [ ] 브랜드명으로 검색했을 때 결과가 과도하게 많거나 엉뚱하지 않다.
- [ ] 제품 종류로 검색하면 관련 제품만 나온다.
- [ ] 검색 결과의 이미지, 연식과 특징만으로 제품을 구분할 수 있다.
- [ ] 검색 결과가 없을 때 일반 제품 등록과 정보 요청 경로가 이해된다.
- [ ] 최근 검색어가 중복 없이 최대 5개로 유지된다.

### 4. 제품 상세와 정보 정확성

- [ ] 제품명, 브랜드, 시리즈와 모델명이 맞다.
- [ ] 공식 자료, 유사 제품 참고, 일반 관리법의 구분이 명확하다.
- [ ] 제품 이미지가 실제 제품과 비슷하다.
- [ ] 이미지가 로딩되지 않아도 화면이 깨지지 않는다.
- [ ] 관리 준비물과 단계가 해당 제품 종류에 맞다.
- [ ] 냉장고, 식기세척기, 세탁기 등 제품별 관리법이 서로 다르다.
- [ ] 위험 행동과 사용 중단 조건이 이해하기 쉽다.
- [ ] 제품 정보 수정 후 별칭과 날짜, 메모가 유지된다.
- [ ] 제품을 다른 공간으로 이동하면 이전 공간에서 사라지고 새 공간에 나타난다.
- [ ] 제품 삭제 후 목록과 최근 본 제품에서 정상적으로 정리된다.

### 5. 관리 기록과 소모품

- [ ] 관리 기록을 추가할 수 있다.
- [ ] 청소, 점검, 소모품 교체 등 기록 유형이 구분된다.
- [ ] 기록 날짜, 메모와 사용 용품이 저장된다.
- [ ] 기록 수정과 삭제가 동작한다.
- [ ] 제품을 삭제해도 과거 기록을 확인할 수 있다.
- [ ] 필터와 소모품을 직접 추가하고 수정할 수 있다.
- [ ] 소모품 교체일과 다음 교체일이 저장된다.
- [ ] 소모품 교체가 일반 청소 기록으로 잘못 계산되지 않는다.
- [ ] 기록이 많아져도 검색과 필터가 사용하기 쉽다.

### 6. 많은 데이터와 재실행

- [ ] 최소 20개 이상 제품을 등록해도 목록 이동이 부드럽다.
- [ ] 최소 50개 제품을 가정해 공간별 탐색이 어렵지 않다.
- [ ] 앱을 완전히 종료하고 다시 열어도 모든 공간과 제품이 유지된다.
- [ ] 에뮬레이터를 재부팅해도 데이터가 유지된다.
- [ ] 제품을 연속으로 빠르게 등록해도 누락이나 중복이 발생하지 않는다.
- [ ] 여러 화면을 오가며 수정해도 이전 값으로 되돌아가지 않는다.
- [ ] 홈 검색으로 등록 제품을 빠르게 찾을 수 있다.
- [ ] 최근 본 제품 순서가 실제 사용 순서와 맞다.

### 7. 오프라인과 오류 복구

- [ ] 비행기 모드에서도 등록 제품과 내장 관리법이 열린다.
- [ ] 비행기 모드에서도 일반 제품 등록이 가능하다.
- [ ] 서버가 없어도 모델 검색이 내장 카탈로그로 대체된다.
- [ ] 외부 이미지가 없어도 제품 상세가 열린다.
- [ ] 공식 링크를 열지 못할 때 앱이 종료되지 않는다.
- [ ] 손상된 데이터가 생겨도 앱이 빈 화면이나 무한 로딩에 빠지지 않는다.

### 8. 백업과 복원

- [ ] 전체 제품 등록 후 백업 코드를 복사한다.
- [ ] 백업 코드가 중간에 잘리지 않았는지 확인한다.
- [ ] 제품 하나를 추가한 뒤 이전 백업으로 복원한다.
- [ ] 복원 후 공간, 제품, 기록과 소모품 수가 맞다.
- [ ] 잘못된 백업 코드를 넣어도 기존 데이터가 유지된다.
- [ ] 전체 삭제 경고와 결과가 명확하다.

## 실제 휴대폰에서만 확인할 체크리스트

### 1. 카메라와 사진

- [ ] 첫 카메라 실행 시 권한 안내가 자연스럽다.
- [ ] 권한을 거부했을 때 다시 설정할 방법을 이해할 수 있다.
- [ ] 어두운 곳, 반사되는 라벨과 작은 글씨를 촬영할 수 있다.
- [ ] 세로·가로 방향 라벨에서 모델명 후보가 나온다.
- [ ] 한글과 영문이 섞인 라벨에서도 모델명이 추출된다.
- [ ] 사진 앱에서 기존 라벨 사진을 선택할 수 있다.
- [ ] 촬영한 사진이 앱 데이터에 불필요하게 남지 않는다.

### 2. QR·바코드

- [ ] 카메라 초점이 빠르게 맞는다.
- [ ] 손전등과 카메라 전환이 동작한다.
- [ ] 설명서 QR 코드에서 유효한 주소나 모델명을 얻는다.
- [ ] 제품 바코드가 단순 숫자일 때 검색 실패 안내가 적절하다.
- [ ] 같은 코드를 연속 인식해 화면이 여러 번 열리지 않는다.

### 3. 외부 앱 연결

- [ ] 공식 설명서가 Safari 또는 기본 브라우저에서 열린다.
- [ ] PDF 설명서를 확대하고 다시 앱으로 돌아올 수 있다.
- [ ] YouTube 링크가 브라우저 또는 YouTube 앱에서 열린다.
- [ ] 서비스센터 전화 버튼이 전화 앱으로 연결된다.
- [ ] 쿠팡 등 구매 링크가 올바른 상품 또는 검색 결과로 열린다.
- [ ] 외부 앱에서 돌아왔을 때 작성 중인 내용이 유지된다.

### 4. 키보드와 화면

- [ ] 한글, 영문, 숫자와 하이픈 입력이 정상이다.
- [ ] 자동완성이나 자동수정이 모델명을 임의로 바꾸지 않는다.
- [ ] 키보드가 버튼과 입력칸을 가리지 않는다.
- [ ] 작은 화면과 큰 글자 설정에서도 버튼이 잘리지 않는다.
- [ ] 화면 회전, 잠금과 해제 후 작성 내용이 유지된다.
- [ ] iPhone의 홈 인디케이터와 상단 안전 영역을 침범하지 않는다.

### 5. 앱 생명주기와 성능

- [ ] 앱을 백그라운드로 보낸 뒤 돌아와도 입력값이 유지된다.
- [ ] 운영체제가 앱을 종료한 뒤 다시 열어도 저장 데이터가 유지된다.
- [ ] 배터리 절약 모드에서도 검색과 저장이 동작한다.
- [ ] 느린 Wi-Fi와 모바일 데이터에서 무한 로딩이 없다.
- [ ] 네트워크를 Wi-Fi에서 모바일 데이터로 바꿔도 앱이 멈추지 않는다.
- [ ] 제품 50개 이상에서 스크롤과 검색이 충분히 빠르다.
- [ ] 저장 공간이 부족할 때 이해 가능한 오류가 나온다.

### 6. 실제 배포 환경

- [ ] 서명된 설치본을 삭제하지 않고 업데이트해도 데이터가 유지된다.
- [ ] TestFlight 또는 Play 내부 테스트 설치가 정상이다.
- [ ] 앱 아이콘, 앱 이름과 시작 화면이 최종 브랜드와 일치한다.
- [ ] 실제 서버 주소로 제품 검색과 정보 요청이 동작한다.
- [ ] 운영 문의 이메일과 정책 링크가 열린다.
- [ ] VoiceOver 또는 TalkBack 읽기 순서가 자연스럽다.

| P1 | OCR이 Latin 스크립트 중심 | 한글 품명·제조사 인식률 저하 | 중간~높음 | 실제 한국 제품 라벨 표본 테스트와 후보 추출 개선 |

| P1 | 공간 삭제 시 내부 제품도 즉시 삭제 | 실수로 대량 데이터 손실 | 중간 | 삭제 전 백업, 실행 취소 또는 보관함 |

| P2 | 동일 모델 중복 판단이 같은 공간 중심 | 다른 공간의 실수 중복 등록을 놓칠 수 있음 | 중간 | 집 전체 중복 후보도 약한 경고로 표시 |
| P2 | 별칭이 같은 제품을 중복으로 판단 | 흔한 별칭 사용 시 불필요한 경고 | 중간 | 별칭 단독 일치의 경고 강도 낮추기 |

| P2 | 서버 오류는 대부분 로컬 대체로 조용히 처리 | 최신 정보가 아닌지 사용자가 모름 | 중간 | 오프라인/내장 데이터 사용 상태 표시 |

## 이번 실사용에서 특히 집중할 항목

1. 제품 등록 20개를 넘긴 뒤에도 원하는 제품을 빠르게 찾을 수 있는가.
2. 모델명을 모르는 제품을 등록할 때 포기하고 싶은 순간이 어디인가.
3. 가구를 등록한 뒤 제공되는 정보가 실제로 도움이 되는가.
4. 같은 브랜드와 비슷한 제품이 많을 때 이미지만으로 구분 가능한가.
5. 등록 후 앱을 다시 실행했을 때 모든 데이터가 그대로인가.
6. 잘못 등록한 제품의 수정, 이동과 삭제가 부담 없이 가능한가.
7. 백업 코드가 실제 전체 데이터를 담고 복원되는가.
8. 사용자가 다시 앱을 열 이유가 제품 설명서, 기록, 소모품 중 무엇인지 확인한다.

## 앱 안에서 문제 남기기

- 제품 정보, 관리법, 링크 문제: 제품 상세의 `정보 오류·불편 신고`
- 버튼, 저장, 화면 동작 문제: 설정의 `앱 기능 문제 신고`
- 과정이 헷갈리거나 번거로운 경우: 설정의 `사용하기 불편한 점`

신고에는 발생 화면과 선택한 제품 정보가 자동으로 포함된다. 자세한 내용에는
수행한 순서, 기대한 결과, 실제 결과를 적고 개인정보는 입력하지 않는다.

## 사용자 피드백 우선순위

- 즉시 수정: 데이터 손실, 앱 종료, 저장 실패, 잘못된 제품 연결
- 다음 수정: 등록 포기, 검색 실패, 중복·공간 혼란, 관리법 불일치
- 다듬기: 문구, 간격, 색상, 이미지 크기와 정렬
- 장기 검토: 계정 동기화, 알림, 제휴 상품과 대규모 서버 카탈로그
