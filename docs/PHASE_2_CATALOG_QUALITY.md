# Phase 2: 제품 카탈로그 품질 강화

상태: 진행 중

Phase 2는 제품 수를 무작정 늘리는 단계가 아니다. 제품 정보가 어디에서 왔고,
어느 수준까지 검수됐는지 추적할 수 있는 기반을 먼저 만든다.

## 완료된 기반 작업

### 제품 데이터 규칙

카탈로그 제품은 다음 정보를 관리한다.

- 내부 제품 ID
- 제품 종류, 표시 이름, 브랜드, 제조사와 모델명
- 처리방식과 주요 스펙
- 관리법 수준과 검수 상태
- 마지막 출처 확인일
- 설명서, 고객지원, 출시 연도, 단종 여부 등의 선택 정보
- 설치 방식과 필터 및 소모품

기존 단일 `sourceTitle`, `sourceUrl` 필드는 이전 앱 데이터와의 호환을 위해
유지한다. 새 카탈로그 데이터는 `sources`를 기준으로 작성한다.

### 다중 출처

제품 하나에 여러 출처를 연결할 수 있다.

각 출처에는 다음 내용을 저장한다.

- 출처 ID
- 제목과 URL
- 출처 유형
- 게시 주체
- 확인일
- 근거로 사용한 정보
- 공식 여부
- 현재 유효 여부

지원 출처 유형:

- 공식 제품 페이지
- 공식 설명서
- 공식 영상
- 공식 고객지원
- 판매 페이지
- 가격비교 스펙
- 설치업체 자료
- 유사 제품 참고
- 일반 제품군 관리법

### 근거 추적

- `specSourceIds`: 스펙 항목과 출처 연결
- `stepSourceIds`: 관리 단계와 출처 연결
- 숫자가 포함된 스펙은 반드시 출처가 연결되어야 한다.
- 존재하지 않는 출처 ID를 참조하면 서버가 카탈로그 로드를 거부한다.

### 검수 이력

`reviewHistory`에 상태, 검수자, 검수일과 메모를 기록한다.

- `draft`: 조사 중이며 공개하지 않음
- `reviewed`: 모델과 안전 범위를 1차 검수함
- `verified`: 공식 자료와 주요 정보를 교차 확인함

공개 상태와 마지막 검수 이력의 상태가 다르면 카탈로그 로드를 거부한다.

### 재검수 판단

다음 출처는 재검수 대상으로 판단할 수 있다.

- `isActive`가 `false`인 출처
- 마지막 확인 후 정해진 기간이 지난 출처

현재 기본 재검수 기준은 180일이다.

## 현재 마이그레이션 상태

- 에코업 DCS-HM4AG-W의 제품 스펙과 공식 영상을 별도 출처로 분리
- 일반 냉장고 관리법을 비공식 제품군 가이드로 명시
- 일반 공기청정기 관리법을 비공식 제품군 가이드로 명시
- 사용자 제품 저장 시 다중 출처 정보 유지
- 제품 상세 화면에서 출처별 공식 여부, 유형, 게시 주체, 확인일과 근거 표시

## 다음 작업

Phase 2의 다음 작업은 카탈로그 확장이다.

1. 제품군별 조사 템플릿 확정
2. 음식물처리기, 냉장고, 세탁기, 에어컨, 공기청정기 순서로 확장
3. 제품군별 일반 관리법 최소 1개 확보
4. 대표 브랜드와 모델 조사
5. 최소 25개의 검수된 카탈로그 항목 확보

제품을 추가할 때는 화면보다 출처와 검수 품질을 먼저 확인한다.
