# 제품 관리 앱 방향성과 로드맵

이 문서는 개발 과정에서 방향이 흔들릴 때 다시 확인하기 위한 기준 문서다.

구체적인 개발 순서, 완료 기준과 다음 스프린트는
`DETAILED_EXECUTION_PLAN.md`에서 관리한다.

기능을 추가하거나 구조를 바꾸기 전에 항상 다음 질문으로 돌아온다.

> 이 기능이 사용자가 자기 제품을 찾고, 믿을 수 있는 관리 정보를 얻고, 관리 이력을 남기는 경험을 더 좋게 만드는가?

답이 명확하지 않다면 지금 만들지 않는다.

---

## 1. 우리가 만들고 있는 앱

이 앱은 매일 청소 일정을 강요하는 앱이 아니다.

우리가 만들고 있는 것은 다음과 같은 서비스다.

> 내가 사용하는 제품을 등록해두면 필요할 때 정확한 관리법, 청소용품, 소모품과 관리 기록을 바로 확인할 수 있는 우리 집 제품 관리 앱

사용자가 매일 앱을 열 필요는 없다.

대신 아래와 같은 순간에 가장 먼저 떠올리는 앱이 되는 것이 목표다.

- 제품 청소법이 기억나지 않을 때
- 냄새, 오염, 소음 등 신경 쓰이는 문제가 생겼을 때
- 필터나 소모품 교체 시기를 확인하고 싶을 때
- 예전에 어떻게 관리했는지 다시 보고 싶을 때
- 제품 모델명, 설명서, 관리 영상을 찾고 싶을 때
- 어떤 청소용품을 사용해야 안전한지 알고 싶을 때

---

## 2. 제품의 핵심 가치

### 제품을 찾을 수 있어야 한다

사용자는 제품명, 브랜드, 모델명으로 제품을 검색할 수 있어야 한다.

모델명을 모르는 사용자도 제품 종류만으로 등록할 수 있어야 한다. 제품 등록 과정에서 막히는 사용자가 없어야 한다.

### 정보 수준을 솔직하게 표시해야 한다

모든 관리 정보를 같은 수준의 사실처럼 보여주지 않는다.

- 모델 전용 공식 정보
- 제조사 공식 영상
- 같은 브랜드 또는 유사 제품 참고 정보
- 제품군에 공통으로 적용되는 일반 관리법
- 사용자가 직접 입력한 정보

정확한 정보가 없을 때는 없다고 말한다. 유사 제품을 참고했다면 그 사실도 분명하게 표시한다.

### 필요할 때 빠르게 답을 얻어야 한다

사용자는 앱을 공부하기 위해 들어오지 않는다.

제품 상세 화면에서 다음 정보를 빠르게 찾을 수 있어야 한다.

- 어떻게 청소하는지
- 무엇을 준비해야 하는지
- 무엇을 하면 안 되는지
- 어떤 소모품이나 청소용품이 필요한지
- 마지막으로 언제 관리했는지
- 공식 설명서나 영상은 어디에 있는지

### 죄책감을 주지 않아야 한다

밀린 청소 개수나 실패한 일정을 강조하지 않는다.

앱은 사용자를 재촉하는 감독자가 아니라, 필요할 때 도와주는 제품 관리 도구여야 한다.

---

## 3. 제품 등록 경험

당분간 가장 중요한 기능은 제품 등록이다.

사용자가 제품을 등록하려다가 브랜드나 모델명을 찾지 못하면 앱의 핵심 가치를 경험하기 전에 이탈할 수 있다.

다음 세 가지 등록 방법을 항상 제공한다.

1. 제품명, 브랜드 또는 모델명으로 검색
2. 브랜드와 대표 모델 목록에서 선택
3. 정보를 모르면 제품 종류만 등록

정확한 모델이 카탈로그에 있다면 해당 정보를 연결한다.

정확한 모델이 없다면 다음 순서로 안내 범위를 넓힌다.

1. 같은 모델의 공식 자료
2. 같은 브랜드의 유사 제품
3. 같은 처리방식 또는 제품군
4. 일반 관리법

그래도 정보를 찾지 못하면 사용자가 제품을 직접 등록하고 정보 추가를 요청할 수 있게 한다.

---

## 4. 제품 정보 수집 원칙

이 앱의 장기 경쟁력은 화면 수가 아니라 제품 데이터의 품질에서 나온다.

제품 정보는 다음 출처를 우선한다.

1. 제조사 공식 홈페이지
2. 공식 사용설명서
3. 제조사 공식 유튜브와 고객지원 자료
4. 공식 판매 페이지
5. 신뢰할 수 있는 가격비교 사이트의 기본 스펙
6. 설치업체 또는 공식 고객센터 안내
7. 같은 구조나 처리방식을 가진 유사 제품 자료

모든 제품 정보에는 가능하면 다음 내용을 기록한다.

- 제품 종류
- 브랜드
- 제조사
- 정확한 모델명
- 처리방식과 주요 스펙
- 관리법의 근거
- 출처 이름과 URL
- 마지막으로 확인한 날짜
- 모델 일치 수준
- 검수 상태
- 검수 담당자와 검수 메모

정보를 많이 모으는 것보다 신뢰 수준을 정확하게 표시하는 것이 중요하다.

---

## 5. 제품 상세 화면

제품 상세 화면은 제품 관리의 중심이다.

장기적으로 다음 정보를 포함한다.

- 제품 기본 정보
- 브랜드, 제조사와 모델명
- 제품 처리방식과 주요 스펙
- 공식 설명서
- 공식 관리 영상
- 평상시 관리법
- 냄새가 날 때 대처법
- 오염이 심할 때 대처법
- 이상 소음이나 누수가 있을 때의 조치
- 하면 안 되는 행동
- 필요한 준비물
- 추천 청소용품
- 필터와 소모품
- 교체 주기
- 관리 이력과 사용자 메모
- AS 연락처와 보증 정보

정보를 한 화면에 무작정 길게 나열하지 않는다.

다음과 같이 목적별로 나누는 방향을 고려한다.

- 관리법
- 문제 해결
- 소모품
- 기록
- 제품 정보

---

## 6. 제품 카탈로그 확대 순서

처음부터 모든 제품을 지원하려 하지 않는다.

사용자가 관리법을 자주 검색하고 반복적으로 관리하는 제품부터 집중한다.

초기 우선순위는 다음과 같다.

1. 음식물처리기
2. 냉장고
3. 세탁기
4. 건조기
5. 에어컨
6. 공기청정기
7. 식기세척기
8. 전자레인지
9. 로봇청소기
10. 정수기

각 제품군에서 인기 브랜드와 대표 모델을 먼저 지원한다.

그 이후에는 사용자의 검색 실패와 직접 등록 데이터를 바탕으로 조사 우선순위를 정한다.

운영자의 추측보다 실제 사용자가 찾는 제품을 우선한다.

---

## 7. 관리 기록

기록 기능은 단순한 청소 완료 체크가 아니다.

제품별 관리 이력으로 발전시킨다.

- 마지막으로 청소한 날짜
- 필터나 소모품을 교체한 날짜
- 사용한 청소용품
- 발생했던 문제
- 관리 후 결과
- 사용자가 남긴 메모
- 다음에 확인할 내용

사용자는 문제가 다시 생겼을 때 이전에 어떤 방법을 사용했는지 확인할 수 있어야 한다.

기록은 사용자를 평가하기 위한 데이터가 아니라, 다음 관리를 더 쉽게 만들기 위한 기억 보조 도구다.

---

## 8. 알림 원칙

매일 해야 할 일을 자동으로 쌓는 구조로 돌아가지 않는다.

알림은 사용자가 원하는 제품에만 직접 설정한다.

- 공기청정기 필터 확인
- 세탁기 통세척
- 음식물처리기 내부 점검
- 에어컨 필터 청소
- 정수기 필터 교체

알림을 놓쳐도 밀린 일정이나 실패로 표시하지 않는다.

앱을 열었을 때 관리해보면 좋은 제품을 부드럽게 제안하는 정도로 표현한다.

---

## 9. 청소용품 추천과 수익화

추천 제품은 단순한 구매 링크가 아니다.

각 추천에는 다음 정보가 필요하다.

- 왜 이 제품이 필요한지
- 어떤 재질과 상황에 사용할 수 있는지
- 사용하면 안 되는 부분은 무엇인지
- 더 저렴하거나 일반적인 대체품이 있는지
- 광고 또는 제휴 상품인지

가능한 수익 모델은 다음과 같다.

- 쿠팡 파트너스 등 제휴 링크
- 필터와 소모품 구매 연결
- 제조사 공식 관리용품 소개
- 브랜드 협찬
- 프리미엄 제품 관리 기능

광고 여부는 반드시 명확하게 표시한다.

광고 때문에 관리 정보의 신뢰도가 흔들리면 앱의 핵심 가치가 사라진다.

---

## 10. 계정과 가족 공유

초기에는 휴대폰 내부 저장으로 충분하다.

실제 서비스 단계에서는 계정과 동기화가 필요하다.

- 휴대폰을 바꿔도 제품 목록 유지
- 관리 기록 백업과 복구
- 가족과 제품 목록 공유
- 여러 집이나 공간 관리
- 설명서와 보증 정보 보관

가족 공유는 업무 배정이나 책임 비교가 아니라 함께 제품을 관리하는 도구여야 한다.

누가 청소하지 않았는지를 강조하지 않는다.

---

## 11. 사용자 제보와 검수

카탈로그에 없는 제품은 사용자가 제보할 수 있게 한다.

제보 항목은 다음과 같다.

- 제품 종류
- 브랜드
- 모델명
- 제품 라벨 사진
- 공식 설명서 링크
- 공식 영상 링크
- 알고 싶은 관리 정보

사용자 제보는 바로 공개하지 않는다.

다음 검수 단계를 거친다.

1. 제보 접수
2. 모델명과 제조사 확인
3. 공식 출처 조사
4. 관리법 작성
5. 주의사항 검토
6. 검수 완료 후 공개

검수 상태는 다음과 같이 관리한다.

- `draft`: 조사 중이며 앱에 노출하지 않음
- `reviewed`: 기본 검수 완료
- `verified`: 모델명과 주요 출처 확인 완료

---

## 12. 커뮤니티 방향

일반적인 SNS 피드는 핵심 기능이 아니다.

커뮤니티를 확장한다면 제품별 관리 경험이 쌓이는 공간으로 만든다.

- 청소 전후 사진
- 특정 모델 관리 후기
- 사용한 청소용품 후기
- 모델별 관리 팁
- 잘못된 관리법 신고
- 공식 정보와 사용자 팁 구분

제품 관리 기능이 충분히 완성되기 전까지 커뮤니티 개발 우선순위는 낮게 둔다.

---

## 13. 문제 진단 기능

카탈로그와 기록이 충분히 쌓이면 문제 상황 중심의 안내로 발전할 수 있다.

- 음식물처리기에서 냄새가 난다
- 세탁기 고무패킹에 곰팡이가 생겼다
- 공기청정기에서 이상한 소리가 난다
- 냉장고 문 주변에 물이 생긴다
- 에어컨에서 냄새가 난다

앱은 몇 가지 질문을 거쳐 다음 중 하나를 안내한다.

- 사용자가 직접 관리 가능
- 공식 설명서 확인 필요
- 소모품 교체 필요
- 제품 사용 중단 권장
- 제조사 또는 설치업체 문의 필요

안전이나 고장과 관련된 문제를 무리하게 청소 방법으로 해결하려 하지 않는다.

---

## 14. 운영자 도구

제품 수가 늘어나면 JSON 파일만으로 관리하기 어려워진다.

그 시점에 운영자용 관리 화면을 만든다.

- 신규 제품 등록
- 제품 정보 수정
- 출처 링크 관리
- 검수 상태 변경
- 오래된 정보 재검토
- 사용자 제보 처리
- 잘못된 관리법 수정
- 광고 상품 표시
- 검색 실패 제품 확인
- 인기 제품과 조회 수 확인

관리 도구를 통해 앱을 다시 배포하지 않고도 제품 정보를 추가하고 수정할 수 있어야 한다.

실제 출시 전에는 기능 추가보다 안정성과 신뢰성을 우선한다.

- 개인정보처리방침
- 이용약관
- 광고 및 제휴 표시 기준
- 제품 관리법 면책 안내
- 계정과 데이터 삭제
- 데이터 백업과 복구
- 오류 및 문의 접수
- Android 권한 최소화
- 앱 이름과 아이콘
- Play Store 등록 자료
- 다양한 실제 기기 테스트

제품 손상이나 안전사고를 막기 위해 공식 설명서 우선 원칙과 사용 중단 기준을 명확히 표시한다.

## 16. 당분간 하지 않을 것

다음 기능은 제품 관리의 핵심 경험이 완성되기 전까지 우선순위에서 제외한다.

- 매일 청소 체크리스트
- 밀린 청소 개수 강조
- 청소 타이머
- 체력이나 남은 시간 선택
- 복잡한 포인트와 레벨 시스템
- 일반적인 SNS 피드
- 출처가 불명확한 AI 자동 청소법
- 검수되지 않은 사용자 정보 즉시 공개
- 제품 데이터보다 앞선 과도한 화면 장식
- 사용 이유가 불분명한 기능 추가

## 17. 큰 개발 순서

1. 제품 검색과 등록 경험 완성
2. 핵심 제품군 카탈로그 확대
3. 제품 상세 정보와 출처 강화
4. 관리 기록과 소모품 교체 기록 개선
5. 서버 데이터베이스 도입
6. 사용자 제보와 운영자 검수 도구 구축
7. 계정과 가족 공유
8. 선택적 알림

## 18. 기능 추가 전 확인할 질문

새로운 아이디어가 생기면 개발하기 전에 다음 질문에 답한다.

1. 이 기능은 실제 사용자가 겪는 어떤 문제를 해결하는가?
2. 제품 등록, 관리 정보, 소모품, 기록 중 어느 핵심 가치와 연결되는가?
3. 이 기능 때문에 제품을 찾거나 관리법을 보는 과정이 복잡해지지 않는가?
4. 사용자가 매일 앱을 열어야만 의미가 있는 기능은 아닌가?
5. 사용자에게 죄책감이나 부담을 주지 않는가?
6. 정보의 출처와 신뢰 수준을 설명할 수 있는가?
7. 지금 반드시 필요한가, 아니면 핵심 기능이 완성된 후에도 되는가?
8. 이 기능을 제거해도 앱의 핵심 가치가 유지된다면 우선순위를 낮춰야 하지 않는가?

## 최종 원칙

> 기능을 많이 만드는 것보다, 사용자가 자기 제품을 찾고 믿을 수 있는 관리법을 얻는 경험을 먼저 완성한다.

이 원칙에서 멀어지는 기능은 만들지 않는다.

앱을 매일 사용하게 만드는 것이 목표가 아니다.

사용자가 제품 관리로 곤란한 순간에 자연스럽게 다시 찾아오는 앱을 만든다.
