# Phase 10: 카탈로그 운영 기반

## 목표

제품 자료를 앱 코드에 직접 덧붙이는 방식에서 벗어나, 작성·검수·공개 이력이
남는 운영 구조를 마련한다.

## 상태 흐름

`draft -> reviewed -> verified`

- `draft`: 작성과 수정 가능, 앱 비공개
- `reviewed`: 출처와 관리 내용 검수 완료, 앱 공개
- `verified`: 출시 기준 최종 확인, 앱 공개
- 공개 제품 수정 시 반드시 `draft`로 되돌린 뒤 재검수

## 구현 내용

- SQLite 운영 카탈로그 저장소
- 관리자 API 키가 설정된 경우에만 관리 API 활성화
- 초안 등록·수정, 상태 변경, 초안 삭제 API
- 작업자, 메모, 시각을 남기는 감사 로그
- 동일 저장소를 다루는 CLI 운영 명령
- 공개 카탈로그와 운영 DB의 검수 완료 제품 자동 병합
- 서버 검색의 모델명 오타 허용

## 출시 전 외부 작업

- 운영 서버에서는 SQLite 대신 관리형 PostgreSQL 적용
- API 키 단일 인증을 관리자 계정과 역할 기반 권한으로 교체
- DB 자동 백업과 복구 훈련
- 관리자 웹 화면 제작
- 작성자와 검수자를 분리하는 운영 규칙 확정
- 50개 출시 제품의 공식 출처와 재검수 주기 등록
