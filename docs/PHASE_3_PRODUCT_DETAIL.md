# Phase 3: 제품 상세 화면 재설계

상태: 핵심 구현 완료

## 이번 구현

제품 상세 화면을 한 개의 긴 목록에서 목적별 탭으로 변경했다.

- 관리법
- 문제 해결
- 소모품
- 기록
- 제품 정보

상단에는 제품 이름, 브랜드와 모델명, 정보 일치 수준, 다음 관리일과 빠른 관리
기록 버튼을 배치했다.

## 관리법

- 제품 요약
- 준비물
- 단계별 관리 순서
- 하면 안 되는 행동
- 공식 또는 참고 영상

관리 순서를 확인하기 위해 제품 스펙과 출처 영역을 지나치게 스크롤하지 않아도 된다.

## 문제 해결

제품군에 따라 자주 발생하는 증상을 구분한다.

- 냄새
- 세척 또는 냉각 성능 저하
- 배수 문제
- 이상 소음
- 누수

사용자가 해결해도 되는 점검과 사용 중단 후 전문가에게 문의해야 하는 상황을
구분한다. 분해나 전기·배관 수리를 사용자 관리법으로 안내하지 않는다.

## 소모품

- 기본 관리 용품
- 용품을 선택할 때 확인할 점
- 특정 추천 제품과 광고 여부

## 기록

현재 제품의 관리 기록만 모아 표시한다. 상단의 빠른 기록 버튼과 기록 탭의
`기록 추가` 버튼은 기존 관리 완료 기록을 저장한다.

기록 유형, 메모, 사진과 수정·삭제는 Phase 4에서 확장한다.

## 제품 정보

- 브랜드와 모델명
- 구매일과 설치일
- 처리 방식과 사용자 메모
- 확인된 스펙
- 관리법의 참고 기준
- 출처별 공식 여부, 게시 주체, 확인일과 근거

## UI 방향

사용자의 성별을 전제하지 않는다.

- 흰색과 밝은 회색을 기본 배경으로 사용
- 차콜 텍스트와 스틸 계열 보조색 사용
- 코랄은 선택 상태와 주요 행동에만 제한적으로 사용
- 마스코트는 홈의 브랜드 요소로만 사용
- 내 제품과 기록 같은 작업 화면에서는 마스코트를 제거
- 장식보다 제품 정보 탐색과 반복 사용 효율을 우선

## 제거된 기능

제품 관리의 핵심 가치와 거리가 있는 `자랑` 탭과 커뮤니티 화면, 데이터 모델과
저장 코드를 제거했다.

## 다음 작업

- 문제 해결 데이터를 카탈로그 모델로 이동
- 필터와 소모품의 교체 주기 구조화
- 공식 설명서와 고객지원 연락처 빠른 행동 추가
- 관리법 난이도와 예상 시간 표시 개선
- 다양한 화면 크기에서 탭과 상단 정보 영역 점검

## 외형 기반 제품 등록

사용자가 정확한 모델명을 모르는 상황을 기본 사용 사례로 본다.

냉장고 등록 흐름:

1. 냉장고 선택
2. 대표 브랜드 선택 또는 직접 입력
3. `이미지와 연식으로 제품 찾기` 실행
4. 4도어, 양문형, 일반형, 김치냉장고 중 비슷한 외형 선택
5. 외형과 출시 시기를 확인해 `이 제품과 비슷해요` 선택

정확한 모델명을 알고 있는 경우에는 브랜드 선택 후 `모델 선택` 버튼을 사용한다.
모델명은 등록 화면에서 직접 타이핑하지 않고 별도 화면에서 처리한다.

- 대표 모델 목록 검색
- 모델 선택
- 목록에 없으면 직접 입력
- 모델명을 모르면 건너뛰기

대표 브랜드 목록에는 일반 제품군 데이터의 `브랜드 미상` 값을 표시하지 않는다.

외형 선택은 정확한 모델 확정이 아니다. 다음 정보로 저장한다.

- 브랜드
- 제품 형태
- 대략적인 출시 시기
- 외형 후보 ID
- 정보 수준: `외형 기반 유사 제품`

모델명은 비워두며 제품군 공통 관리법을 연결한다. 필터 규격, 분해 방법이나 전용
부품처럼 모델 차이가 큰 정보는 정확한 모델이 확인되기 전까지 제공하지 않는다.

현재 후보 카드는 제품 형태를 구분하기 위한 대표 도식을 사용한다. 이후 제조사
공식 이미지와 사용 권한 및 출처가 확보되면 같은 카드 구조에 공식 이미지를
연결한다.
