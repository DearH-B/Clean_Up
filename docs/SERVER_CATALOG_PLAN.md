# 제품 카탈로그 서버 운영 계획

## 목표

Flutter 앱에 브랜드별 제품 자료를 직접 넣지 않고 서버에서 검색, 등록, 갱신할 수 있게 한다.

## 현재 구현

- 제품 검색: `GET /v1/products`
- 제품 상세: `GET /v1/products/{product_id}`
- 카테고리별 브랜드: `GET /v1/brands?category=TV`
- 브랜드별 모델: `GET /v1/models?category=TV&brand=삼성전자`
- 서버 연결 실패 시 앱 내장 목록으로 자동 전환
- 모델 후보에 모델명, 표시명, 출시연도, 이미지 URL, 공식 제품 URL 저장 가능

## 데이터 구분

`products.json`은 관리법과 출처 검수가 끝난 제품을 저장한다.

`models.json`은 제품 등록 화면에서 사용자가 제품을 식별하기 위한 후보를 저장한다. 모델이 여기에 있다고 해서 모델 전용 관리법까지 검수되었다는 의미는 아니다.

## 수집 원칙

1. 제조사 공식 제품 페이지와 공식 고객지원 자료를 우선한다.
2. 모델명, 출시연도, 대표 이미지, 공식 URL과 확인일을 저장한다.
3. 판매처 자료는 보조 출처로만 사용한다.
4. 이미지 파일을 직접 저장하거나 노출하기 전에 이용 조건을 확인한다.
5. 자동 수집 데이터는 `draft`로 저장하고 사람의 검수 후 공개한다.
6. 삭제되거나 변경된 페이지는 정기 재검수 대상으로 표시한다.

## 다음 확장

1. JSON 저장소를 PostgreSQL로 교체
2. 브랜드, 제품군, 모델, 출처, 관리법을 별도 테이블로 분리
3. 관리자용 제품 검수 화면 구축
4. 공식 페이지 수집 작업과 변경 감지 작업 추가
5. 바코드, QR, 시리얼 번호를 모델과 연결
6. 앱에서 검색 결과의 출처와 정확도 표시 강화

## 첫 데이터 범위

삼성전자와 LG전자의 TV, 냉장고부터 시작한다. 각 제품군에서 최신 제품과 사용자가 많이 보유한 이전 연식 제품을 함께 수집한다.
