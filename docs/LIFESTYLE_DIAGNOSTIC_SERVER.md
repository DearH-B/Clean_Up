# 생활 문제 데이터 서버 운영

## 구조

- 앱은 `GET /v1/diagnostics?productType=세면대`를 먼저 조회한다.
- 서버에 검수된 데이터가 있으면 해당 버전을 사용한다.
- 서버 연결 실패, 잘못된 응답, 빈 결과일 때는 앱 내장 안내를 사용한다.
- 따라서 오프라인 상태에서도 문제 해결 기능은 사라지지 않는다.

## 데이터 위치

- 서버 원본: `server/data/product_diagnostics.json`
- 서버 로더: `server/app/diagnostics.py`
- Flutter 저장소: `lib/repositories/product_diagnostic_repository.dart`
- 앱 내장 폴백: `lib/data/product_diagnostics.dart`

## 배포 원칙

새 제품 종류는 안전 문구, 대처 순서, 도구, 중단 신호와 검수 상태를 확인한 뒤
서버 JSON에 추가한다. 서버에 등록되지 않은 종류는 기존 앱 안내를 유지하므로,
검수되지 않은 간단한 데이터로 상세 안내를 덮어쓰지 않는다.

현재 앱에서 등록 가능한 표준 제품 종류 전체가 서버 데이터로 전환됐다.

- 물 사용 설비와 정수기
- 주방 가전과 음식물처리기
- 냉장고·김치냉장고, 세탁기, 식기세척기
- 공기 관리 제품, 건조기와 청소기
- 패브릭, 전자기기, 창호와 가구

앱 내장 안내는 서버 장애와 오프라인 상황을 위한 안전 폴백으로 유지한다.
`version`은 데이터 변경 배포마다 증가시킨다.

## 자동 품질 검사

서버 테스트는 모든 표준 제품 종류에 대해 다음 조건을 검사한다.

- 최소 3개 이상의 문제 안내
- `reviewed` 검수 상태
- 범용 `generic-*` 진단 미사용
- 단계와 도구가 있는 실행 가능한 안내
- 사용 중단 또는 전문가 문의가 필요한 위험 결과

## 공식 자료 승격

`verified`는 표시용 값이 아니다. 서버는 다음 조건을 만족하지 않으면 시작 과정에서
데이터를 거부한다.

- 공식 사용설명서 출처가 한 건 이상 존재
- 출처 ID가 실제 출처 목록에 존재
- 단계가 있는 안내는 공식 설명서 근거 ID를 연결
- 마지막 검수 이력이 `verified` 상태와 일치

현재 공식 자료 확인 제품군은 냉장고·김치냉장고, 세탁기, 식기세척기다.
음식물처리기를 포함한 나머지 제품군은 공식 자료가 충분히 확보될 때까지
`draft` 상태를 유지한다. 출처가 없는 안내를 `reviewed`로 올리면 서버가
데이터를 거부한다.
