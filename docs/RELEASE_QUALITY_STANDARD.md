# 실제 출시 품질 기준

상태: 적용 중

이 프로젝트의 완료 기준은 더 이상 MVP 동작 여부가 아니다. 사용자가 실제 제품 관리
정보를 믿고 사용할 수 있고, 앱 운영자가 오류를 발견하고 수정할 수 있는 상태를
출시 가능 상태로 본다.

## 제품 데이터 출시 기준

제품 하나를 `출시 검증 완료`로 판단하려면 다음 조건을 모두 충족해야 한다.

1. 제조사, 제품군, 모델 코드와 표시 이름을 공식 자료로 확인한다.
2. 공식 제품 페이지와 공식 설명서 원문을 각각 연결한다.
3. 앱의 모든 스펙과 관리 단계가 어느 출처에 근거하는지 추적할 수 있다.
4. 모델에 없는 기능, 확인되지 않은 부품번호와 임의 관리 주기를 표시하지 않는다.
5. 안전 관련 문구는 설명서 원문과 대조하고 위험한 축약이나 일반화를 하지 않는다.
6. 단종, 설명서 개정, 링크 만료를 재검수할 날짜와 절차가 있다.
7. 서버 데이터와 앱 내장 대체 데이터가 같은 내용을 제공한다.
8. 기존 사용자가 저장한 제품도 최신 검수 데이터로 안전하게 갱신된다.
9. 네트워크 연결·차단 상태에서 각각 정상 동작하고 출처 장애 시 대체 안내가 나온다.
10. 실제 Android 기기에서 등록, 조회, 링크 열기, 기록 저장과 앱 재실행을 검증한다.

## 앱 출시 기준

- 초기 설치, 업데이트, 데이터 마이그레이션과 재설치 흐름 검증
- 주요 화면의 작은 화면, 큰 글자, 화면 회전과 접근성 검증
- 오프라인, 느린 네트워크, 서버 오류와 손상된 응답 처리
- 개인정보 처리방침, 이용약관, 정보 출처와 책임 범위 제공
- 오류 수집, 서버 모니터링, 백업과 복구 절차 준비
- 출시용 서명, 패키지명, 버전 정책과 스토어 등록 자료 준비
- 실제 사용자 시나리오 기반 회귀 테스트와 출시 전 점검표 통과

## 상태 용어

- `조사 중`: 자료 수집 또는 사실 확인이 끝나지 않음
- `데이터 검수 완료`: 공식 자료와 앱 데이터의 내용 검증이 끝남
- `출시 검증 중`: 실제 기기, 네트워크, 운영 및 법적 항목을 점검 중
- `출시 검증 완료`: 제품 데이터와 앱 출시 기준을 모두 통과함

`verified` 카탈로그 상태는 공식 자료 기반 데이터 검수를 의미한다. 이 값만으로 실제
출시 검증이 완료됐다고 판단하지 않는다.
