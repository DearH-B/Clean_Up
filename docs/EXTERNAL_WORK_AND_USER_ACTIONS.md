# 외부 의존 작업과 사용자 준비

코드만으로 안전하게 끝낼 수 없는 작업을 한곳에서 관리한다.

## 1. Phase 1.5: 실제 카메라와 iPhone 검증

현재 할 수 없는 이유:

- iPhone 앱 빌드와 설치에는 macOS와 Xcode가 필요하다.
- 카메라 OCR 품질은 실제 제품 라벨과 실제 기기에서 확인해야 한다.

사용자가 준비할 것:

1. Apple Silicon 또는 Intel Mac
2. 최신 Xcode
3. Apple ID
4. iPhone 16과 USB 케이블 또는 동일 네트워크
5. 테스트할 실제 제품 라벨 10개 이상

재개 문서: `IOS_DEVICE_TESTING.md`, `PHASE_1_5_DEFERRED_VALIDATION.md`

추가 기술 확인:

- 현재 `mobile_scanner 7.2.0`은 APK 빌드에는 성공하지만 Flutter가 향후 Kotlin
  Gradle Plugin 적용 방식 변경 경고를 표시한다.
- 새 버전의 `mobile_scanner`가 Built-in Kotlin을 지원하면 의존성을 올리고
  Android 카메라를 다시 검증해야 한다.

## 2. Phase 2: 검수 카탈로그 25개

현재 자동 완료할 수 없는 이유:

- 2026-06-13 기준 앱 내장 카탈로그는 22개이며 Phase 2 목표까지 3개,
  첫 출시 목표 50개까지 28개가 남았다.
- 삼성 가전 정확 모델 21개는 공식 제품 페이지와 사용자 매뉴얼까지 검수 완료했다.
- 제품 모델별 공식 설명서, 지원 페이지, 출시 연도와 소모품 호환성을 사람이
  교차 확인해야 한다.
- 출처가 부족한 제품을 임의로 `verified`로 만들면 앱의 핵심 신뢰를 해친다.

사용자가 준비하거나 결정할 것:

1. 우선 조사할 브랜드 순서
2. 공식 자료가 없는 판매 종료 모델을 포함할지 여부
3. 데이터 검수자 이름 또는 운영자 계정
4. 제품 이미지 사용 권한 정책

현재 수치 확인:

```powershell
cd server
python manage.py catalog
```

## 3. Phase 7: PostgreSQL과 관리자 웹

현재 보류 이유:

- 로드맵의 DB 전환 기준인 검수 제품 100개, 출처 300개, 운영자 2명 등에 아직
  도달하지 않았다.
- 실제 배포 서버, 데이터베이스 사업자와 관리자 인증 방식이 정해지지 않았다.

사용자가 결정할 것:

1. 서버 제공자: 예) Render, Railway, Fly.io, AWS
2. PostgreSQL 제공자와 월 예산
3. 이미지 저장소: 예) S3, Cloudflare R2
4. 관리자 로그인 방식
5. 백업 보관 기간

그 전까지는 `server/manage.py`로 카탈로그 보고서와 제보 상태를 관리할 수 있다.

## 4. Phase 8: 로그인, 동기화와 가족 공유

현재 보류 이유:

- Google, Apple, 카카오 개발자 앱과 OAuth 키가 없다.
- 계정 데이터 보관 지역, 탈퇴와 삭제 정책이 정해지지 않았다.
- 동기화 서버와 충돌 정책이 아직 없다.

사용자가 준비할 것:

1. 서비스용 도메인과 이메일
2. Google Cloud 프로젝트
3. Apple Developer 계정
4. 카카오 Developers 앱
5. 개인정보처리방침 초안
6. 가족 공유의 소유자·편집자 권한 결정

## 5. Phase 9: 실제 알림

현재 보류 이유:

- Android와 iOS 실제 기기에서 권한 요청, 시간대, 재부팅 후 예약 복구를 검증해야
  한다.
- iOS 검증 환경이 아직 없다.

사용자가 결정할 것:

1. 알림을 제품별로 기본 끔 상태로 둘지
2. 정확한 날짜와 느슨한 기간 안내 중 어느 방식을 쓸지
3. 알림 문구와 야간 금지 시간

## 6. Phase 10: 제휴 상품

현재 보류 이유:

- 쿠팡 등 제휴 프로그램 계약과 표시 문구가 정해지지 않았다.
- 가격, 재고와 링크 유효성을 주기적으로 확인할 운영 방식이 없다.

사용자가 준비할 것:

1. 사용할 제휴 사업자
2. 사업자 가입과 승인
3. 광고 표시 문구
4. 가격 확인 주기
5. 반품·품절 링크 처리 기준

앱은 이미 광고 여부, 추천 이유와 구매 URL을 분리 저장하고 표시한다.

## 7. Phase 12: 출시

현재 코드에서 완료한 안전 기반:

- 주요 로컬 목록 저장 시 직전 데이터 자동 백업
- 현재 저장값이 손상된 경우 백업본 자동 복구
- Flutter 정적 분석, 앱 테스트와 서버 테스트
- Android 에뮬레이터 스모크 테스트

사용자가 준비할 것:

1. 정식 앱 이름
2. 서비스 도메인과 문의 이메일
3. 개인정보처리방침과 이용약관
4. 앱 아이콘 최종안
5. Play Console 개발자 계정
6. App Store 출시 시 Apple Developer 유료 계정
7. 오류 수집 서비스 선택

이 준비가 끝나기 전까지 내부 테스트 APK는 만들 수 있지만 정식 서비스 출시로
간주하지 않는다.
