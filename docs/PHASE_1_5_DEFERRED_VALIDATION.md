# Phase 1.5: 환경 의존 작업과 실기기 검증

상태: 보류

이 단계는 Phase 1의 핵심 등록 흐름과 분리한다. 카메라 실기기, iPhone, 사진
저장 정책 또는 운영 서버가 준비됐을 때 필요한 항목만 선택해 진행한다.

## 1. 제품 라벨 사진과 OCR

목표:

- 제품 라벨을 촬영하거나 사진에서 선택한다.
- 사진에서 브랜드와 모델명 후보를 추출한다.
- 사용자가 후보를 확인하고 수정한 뒤 제품 검색으로 이어간다.

진행 전 결정:

- 사진을 기기에 저장할지 즉시 폐기할지
- 서버 전송 여부와 사용자 동의 문구
- 촬영 이미지의 보관 기간
- OCR 결과가 틀렸을 때의 수정 흐름

완료 조건:

- 실제 Android와 iPhone 카메라에서 촬영이 동작한다.
- OCR 결과를 자동 확정하지 않고 사용자가 확인할 수 있다.
- 사진 권한 거부와 인식 실패 상황에서 직접 입력으로 이어진다.

## 2. 제품 정보 요청 운영 기능

목표:

- 앱에 저장된 검색 실패 요청을 사용자 동의 후 서버로 전송한다.
- 운영자가 요청 빈도와 제품 정보를 확인할 수 있다.
- 처리 상태를 `접수`, `조사 중`, `등록 완료`, `보류`로 관리한다.

진행 전 결정:

- 로그인 없이 요청할 수 있는 범위
- 중복 요청 집계 기준
- 사진과 메모의 개인정보 처리
- 관리자 인증과 접근 권한

완료 조건:

- 같은 검색어와 모델의 중복 요청이 집계된다.
- 운영자가 요청을 카탈로그 조사 작업으로 전환할 수 있다.
- 사용자 동의 없이 로컬 요청이나 사진이 서버로 전송되지 않는다.

## 3. iPhone 16 실기기 검증

Mac과 Xcode를 준비한 뒤 `IOS_DEVICE_TESTING.md`의 순서로 진행한다.

검증 항목:

- 앱 설치와 실행
- QR 코드와 바코드 카메라 스캔
- 카메라 권한 허용 및 거부
- 한국어와 영문 키보드 입력
- 작은 화면에서 글자와 버튼 잘림
- 앱 재실행 후 제품 데이터 유지

## 재개 조건

다음 중 하나가 준비되면 Phase 1.5의 해당 항목만 재개한다.

- 테스트 가능한 Android 실기기
- macOS와 Xcode를 사용할 수 있는 Mac
- 사진 저장 및 서버 전송 정책 결정
- 운영 API와 관리자 화면을 개발할 필요가 생김
