# Phase 1: 제품 검색과 등록 (완료)

상태: 2026-06-09 완료

실기기, 사진 처리 또는 운영 환경이 필요한 후속 작업은
`PHASE_1_5_DEFERRED_VALIDATION.md`로 분리했다. 해당 작업은 Phase 1 완료를
막지 않으며 필요한 환경이 준비됐을 때 진행한다.

## 구현된 흐름

제품 등록은 독립된 전체 화면에서 다음 네 단계로 진행한다.

1. 제품 찾기
2. 제품 확인
3. 내 제품 정보
4. 등록 완료

뒤로 이동해도 현재 입력값은 유지된다. 키보드가 열린 상태에서도 다음 단계 버튼을
사용할 수 있다.

## 검색

- 제품 및 설명서의 QR 코드와 바코드 스캔
- 정확한 모델명
- 하이픈과 공백을 제거한 모델명
- 영문 대소문자 차이
- 브랜드, 제품 종류, 별칭과 줄임말
- 최근 검색
- 자주 등록하는 제품 종류

앱은 서버가 반환한 결과도 다시 정렬하여 정확한 모델명 일치를 가장 먼저 보여준다.
서버 연결에 실패하면 기존 로컬 카탈로그를 사용한다.

스캔한 값이 URL이면 `model`, `modelName`, `product`, `gtin`, `ean`, `upc`
등의 쿼리 값을 우선 검색어로 사용한다. 해당 값이 없으면 URL의 마지막 경로 또는
코드 원문으로 검색한다. 검색 결과가 없어도 기존 직접 등록 흐름을 계속 사용할 수
있으며, 스캔 원문과 코드 형식은 등록 제품에 함께 저장한다.

검색 결과에는 제품명, 브랜드, 모델명, 제품 종류, 검수 상태, 일치 수준, 출처
확인일을 표시한다.

## 사용자 제품 정보

카탈로그 정보와 별도로 다음 정보를 사용자 제품에 저장한다.

- 공간
- 별칭
- 구매일
- 설치일
- 메모

같은 모델을 여러 대 등록할 수 있으며 별칭으로 구분한다.

## 중복 처리

선택한 공간 안에서 다음 조건을 확인한다.

- 같은 카탈로그 제품 ID
- 정규화한 모델명이 같음
- 별칭이 같음

중복을 발견해도 등록을 막지 않는다. 기존 제품을 열거나 별도 제품으로 계속 등록할
수 있다. 다른 공간에 있는 같은 모델은 중복으로 취급하지 않는다.

## 검색 실패

검색 결과가 없을 때 다음 선택지를 제공한다.

- 제품 종류만 등록
- 브랜드와 모델명 직접 입력
- 제품 정보 추가 요청

검색어와 요청 시각, 직접 입력한 제품 정보, 일반 제품 등록 여부는 현재 기기에만
저장한다. 사용자 동의 없이 서버로 전송하지 않는다.

## 완료 판단

- 모델명, 브랜드, 제품 종류로 검색할 수 있다.
- QR 코드와 바코드를 스캔해 검색을 시작할 수 있다.
- 제품을 찾지 못해도 직접 등록하거나 정보 추가 요청을 남길 수 있다.
- 카탈로그 제품과 사용자 입력 정보를 구분해 저장한다.
- 중복 가능성을 안내하면서 같은 모델 여러 대의 등록을 허용한다.
- 등록 과정에서 이전 단계로 돌아가도 입력값이 유지된다.
- Android 빌드, 정적 분석과 자동화 테스트가 통과한다.
