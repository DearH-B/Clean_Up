# Phase 1: 제품 검색과 등록

## 구현된 흐름

제품 등록은 독립된 전체 화면에서 다음 네 단계로 진행한다.

1. 제품 찾기
2. 제품 확인
3. 내 제품 정보
4. 등록 완료

뒤로 이동해도 현재 입력값은 유지된다. 키보드가 열린 상태에서도 다음 단계 버튼을
사용할 수 있다.

## 검색

- 정확한 모델명
- 하이픈과 공백을 제거한 모델명
- 영문 대소문자 차이
- 브랜드, 제품 종류, 별칭과 줄임말
- 최근 검색
- 자주 등록하는 제품 종류

앱은 서버가 반환한 결과도 다시 정렬하여 정확한 모델명 일치를 가장 먼저 보여준다.
서버 연결에 실패하면 기존 로컬 카탈로그를 사용한다.

검색 결과에는 제품명, 브랜드, 모델명, 제품 종류, 검수 상태, 일치 수준, 출처
확인일을 표시한다.

## 사용자 제품 정보

카탈로그 정보와 별도로 다음 정보를 사용자 제품에 저장한다.

- 공간
- 별칭
- 구매일
- 설치일
- 메모

같은 모델을 여러 대 등록할 수 있으며 별칭으로 구분한다.

## 중복 처리

선택한 공간 안에서 다음 조건을 확인한다.

- 같은 카탈로그 제품 ID
- 정규화한 모델명이 같음
- 별칭이 같음

중복을 발견해도 등록을 막지 않는다. 기존 제품을 열거나 별도 제품으로 계속 등록할
수 있다. 다른 공간에 있는 같은 모델은 중복으로 취급하지 않는다.

## 검색 실패

검색 결과가 없을 때 다음 선택지를 제공한다.

- 제품 종류만 등록
- 브랜드와 모델명 직접 입력
- 제품 정보 추가 요청

검색어와 요청 시각, 직접 입력한 제품 정보, 일반 제품 등록 여부는 현재 기기에만
저장한다. 사용자 동의 없이 서버로 전송하지 않는다.

## 남은 Phase 1 작업

- 제품 라벨 사진 촬영 및 선택
- 검색 실패 요청을 운영자가 집계하는 서버 API와 관리자 화면

사진 권한과 파일 보관 정책, 서버 전송 동의가 함께 설계되어야 하므로 위 두 항목은
별도 작업으로 마무리한다.
