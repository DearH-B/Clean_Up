# Phase 4: 제품별 관리 기록

상태: 핵심 구현 완료

완료일: 2026-06-11

## 목표

단순한 관리 완료 한 줄을 제품별 관리 이력으로 확장한다.

사용자는 제품에 문제가 다시 생겼을 때 언제 무엇을 했고 결과가 어땠는지 확인할
수 있어야 한다.

## 기록 유형

- 청소
- 점검
- 필터 교체
- 소모품 교체
- 문제 발생
- AS
- 기타 메모

## 기록 정보

- 제품과 공간
- 관리 날짜와 시간
- 소요 시간
- 사용한 용품
- 문제 증상
- 관리 결과
- 사용자 메모
- 다음 확인일
- 사진 경로를 위한 데이터 필드

이전 앱에서 저장된 기록에는 유형 필드가 없다. 이런 기록은 자동으로 `청소`
유형으로 읽어 기존 데이터를 유지한다.

## 제품 상세 기록

- 현재 제품의 기록만 최신순으로 표시
- `오늘 청소` 버튼 한 번으로 날짜 기록
- `기록 추가`는 날짜와 메모만 먼저 표시
- 유형, 시간, 용품과 결과는 선택 상세 영역에 배치
- 기록 선택 또는 메뉴를 통한 수정
- 삭제 전 확인 대화상자
- 최신 관리 기록을 기준으로 마지막 관리일과 다음 확인일 재계산

문제 발생, AS와 기타 메모 기록은 마지막 관리일을 변경하지 않는다.
필터와 소모품 교체 기록도 마지막 청소일과 분리한다.

## 전체 기록

- 제품명과 메모, 증상, 결과, 사용 용품 검색
- 제품 필터
- 공간 필터
- 기록 유형 필터
- 월별 그룹
- 기록 상세 보기
- 수정과 삭제

수정 또는 삭제 후 제품의 마지막 관리일과 다음 확인일도 함께 다시 계산한다.

## 사진 첨부

사진 경로 필드는 마련했지만 실제 촬영과 갤러리 선택은 후속 작업으로 둔다.

구현 전에 다음 기준을 확정해야 한다.

- 앱 전용 저장소로 사진 복사
- 원본 사진 삭제 시 기록 사진 유지 여부
- 앱 삭제와 데이터 백업 정책
- Android와 iOS 사진 권한
- 이미지 크기 축소와 저장 용량 제한
