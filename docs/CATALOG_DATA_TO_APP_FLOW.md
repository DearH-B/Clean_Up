# 조사 데이터가 앱에 등록되는 과정

## 1. 조사 원본

조사한 제품은 먼저 `research/products/`에 저장한다.

```text
research/products/representative-appliances-10/
├─ research.json
└─ notes.md
```

이 단계의 자료는 앱 사용자가 직접 보지 않는다. 공식 링크, 식별 정확도, 조사 한계와
검수 상태를 보관하는 내부 근거 자료다.

## 2. 공개 카탈로그 변환

검수된 내용만 두 곳에 반영한다.

- 앱 내장 카탈로그: `lib/data/product_catalog.dart`
- 서버 카탈로그: `server/app/representative_catalog.py`

앱 내장 카탈로그는 인터넷이나 서버가 꺼져 있어도 기본 검색을 제공한다. 서버
카탈로그는 나중에 자료를 갱신하거나 더 많은 제품을 제공할 때 사용한다.

두 카탈로그에는 다음 내용이 들어간다.

- 제품 ID
- 제품군, 브랜드, 제조사와 모델명
- 모델 일치 또는 제품군 기준 여부
- 관리 방법, 주기, 준비물과 주의사항
- 공식 출처와 앱 편집 가이드 출처
- 마지막 확인일과 검수 상태

## 3. 앱 검색

사용자가 제품 추가 화면에서 `세탁기`, `WF25CB8895BW`, `삼성 TV`처럼 검색하면
앱은 먼저 서버 카탈로그를 조회한다.

```text
제품 검색
  -> 서버 GET /v1/products
  -> 서버 연결 실패 시 앱 내장 카탈로그 검색
  -> 검색 후보 표시
```

모델 후보 화면은 `GET /v1/models`와 `server/data/models.json`을 사용한다.

## 4. 사용자 제품으로 등록

검색 결과를 선택하면 공개 카탈로그 항목이 `ZoneItem`이라는 사용자 제품 데이터로
변환된다.

```text
ProductCatalogEntry
  -> toZoneItem()
  -> 사용자가 선택한 공간 ID와 별칭 추가
  -> SharedPreferences에 로컬 저장
```

이때 사용자의 구매일, 설치일, 별칭과 메모는 개인 데이터로 저장된다. 공식 카탈로그
원본과 섞이지 않는다.

## 5. 제품 상세 화면

등록된 제품을 열면 다음 탭에서 카탈로그 정보가 사용된다.

- 관리법: 준비물, 순서와 금지 행동
- 문제 해결: 제품군별 기본 대응
- 소모품: 필터와 교체 부품
- 기록: 사용자가 남긴 관리 이력
- 제품 정보: 모델명, 스펙, 출처와 확인일

모델 단위 제품은 `모델명 일치`, 시리즈 단위 제품은 `시리즈 기준`으로
표시한다.

## 6. 카탈로그 갱신

같은 제품 ID의 더 최신 자료가 앱에 포함되면 기존 등록 제품도 갱신한다.

```text
catalogProductId가 같음
  + 새 sourceCheckedAt이 더 최신
  -> 관리 정보와 출처 갱신
  -> 사용자 별칭, 구매일, 메모와 기록은 유지
```

공식 주기가 확인되지 않은 제품은 임의의 다음 관리일을 만들지 않는다.

## 이번 10종의 등록 수준

- 모델 단위: 세탁기 `WF25CB8895BW`, TV `KQ65QNF70AFXKR`,
  전자레인지 `MS23C3535AK`
- 제품군 단위: 냉장고, 에어컨, 무선청소기, 건조기, 공기청정기,
  식기세척기, 김치냉장고

후속 조사에서 제품군 단위 항목에 모델 라벨, 설명서와 부품번호가 확보되면 모델 단위
항목을 추가한다.
