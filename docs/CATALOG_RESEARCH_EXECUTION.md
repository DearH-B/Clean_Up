# 대표 제품 자료 조사 운영안

## 목표

전체 제품을 한꺼번에 수집하지 않는다. 대표 제품 10개를 통해 앱의 조사, 검수,
검색, 제품 식별 구조가 실제로 작동하는지 먼저 검증한다.

## 조사량

예상 작업량은 다음과 같다.

- 복잡한 가전 6개: 제품당 약 4~8시간
- 가구 2개: 제품당 약 2~4시간
- 소모품 2개: 제품당 약 1~3시간
- 출처 재확인과 카탈로그 변환: 전체 약 8~16시간

첫 10개를 제대로 완성하는 데 약 40~80시간을 예상한다. 공식 자료의 접근성,
모델별 설명서 품질, 이미지 이용 조건에 따라 달라질 수 있다.

## 범위를 줄이는 방법

1. 제품군마다 대표 모델 1개만 선정한다.
2. 같은 브랜드의 비슷한 모델을 별도 제품으로 중복 조사하지 않는다.
3. 모델 공통 관리법과 모델 전용 정보를 분리한다.
4. 공식 자료가 없는 항목은 추측하지 않고 미확인으로 둔다.
5. 앱에 당장 표시하지 않는 상세 스펙은 우선순위를 낮춘다.
6. 사진 인식용 이미지는 직접 저장하기보다 공식 URL과 식별 특징을 먼저 확보한다.

## 한 제품의 완료 순서

1. 정확한 모델과 조사 범위를 확정한다.
2. 공식 제품 페이지, 설명서, 고객지원 자료를 찾는다.
3. 모델명 변형, 라벨 문구, 바코드 등 식별 단서를 기록한다.
4. 관리 순서, 주기, 금지 행동, 소모품을 출처별로 정리한다.
5. 자료 간 충돌과 빠진 정보를 기록한다.
6. 제품 복잡도에 맞는 품질 기준으로 검수한다.
7. 검수된 내용만 서버 카탈로그 형식으로 변환한다.
8. 앱 검색과 제품 상세 화면에서 결과를 확인한다.

## 첫 번째 파일럿

첫 번째 조사는 기존 데이터가 있는 `에코업 DCS-HM4AG-W`를 새 템플릿으로
재작성한다. 이 작업으로 조사 파일에서 서버 카탈로그로 옮기는 과정과 누락 필드를
확인한다.

그다음 냉장고 대표 모델을 조사한다. 냉장고는 공식 설명서, 모델 라벨, 내부 관리,
소모품과 주의사항을 모두 시험할 수 있어 사진·OCR 등록 구조를 검증하기 좋다.

## 저장 원칙

- 조사 초안: `research/products/<product-id>/research.json`
- 판단 메모: `research/products/<product-id>/notes.md`
- 공개 카탈로그: `server/data/products.json`
- 모델 검색 후보: `server/data/models.json`
- 공통 품질 기준: `docs/PRODUCT_DATA_QUALITY_STANDARD.md`

원본과 공개 데이터를 분리해 조사 중인 내용이 앱 사용자에게 노출되지 않도록 한다.
