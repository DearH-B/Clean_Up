# 제품 조사 저장소

이 폴더는 공식 자료를 조사하고 검수하기 위한 작업 공간이다.
앱과 서버는 이 폴더의 초안 데이터를 직접 사용하지 않는다.

## 데이터 흐름

```text
공식 자료 조사
  -> research/products에 근거와 초안 저장
  -> 품질 기준에 따라 검수
  -> server/data/products.json과 models.json에 반영
  -> 앱에서 서버 카탈로그 조회
```

## 폴더 구조

```text
research/
├─ README.md
├─ queue.json
├─ templates/
│  └─ product-research.template.json
└─ products/
   └─ <product-id>/
      ├─ research.json
      ├─ notes.md
      ├─ documents/
      └─ images/
```

제품 폴더에는 저작권과 이용 조건을 확인한 파일만 저장한다. 공식 설명서나 이미지의
재배포가 불분명하면 파일을 복사하지 않고 `research.json`에 원문 URL, 문서 제목,
페이지 번호와 확인일만 기록한다.

## 조사 범위 관리

- 첫 목표는 대표 제품 10개다.
- 동시에 조사 중인 제품은 최대 2개로 제한한다.
- 공식 자료를 확보하지 못한 제품은 억지로 완성하지 않고 `blocked`로 표시한다.
- 한 제품을 `reviewed`로 만들기 전에 비슷한 모델을 계속 추가하지 않는다.
- 제품별 예상 조사 시간은 단순 소모품 1~2시간, 가구 2~4시간, 복잡한 가전
  4~8시간으로 본다.

## 상태

- `candidate`: 조사 후보
- `researching`: 공식 자료 조사 중
- `drafted`: 조사 초안 작성 완료
- `reviewing`: 출처와 안전 정보 검수 중
- `reviewed`: 기본 검수 완료
- `published`: 서버 카탈로그 반영 완료
- `blocked`: 공식 자료 부족, 접근 제한 또는 식별 불가

## 파일 역할

### research.json

출처, 식별 단서, 관리법 초안, 소모품, 검수 상태를 구조화해서 기록한다.
사진·OCR·바코드 검색에 필요한 식별 데이터도 이 파일에 보관한다.

### notes.md

자료 간 충돌, 판단 근거, 앱에서 표현할 때의 주의점처럼 구조화하기 어려운 내용을
기록한다. 원문을 길게 복사하지 않고 요약과 위치 정보만 남긴다.

### documents

저장이 허용되는 공식 설명서와 공개 문서를 보관한다. 저장 권한이 불분명하면 비워둔다.

### images

사용 권한이 확인된 이미지, 직접 촬영한 이미지 또는 내부 검수용 이미지만 보관한다.
공식 페이지 이미지는 URL만 기록하는 것을 기본으로 한다.

## 공개 조건

`docs/PRODUCT_DATA_QUALITY_STANDARD.md`의 공통 최소 기준을 통과한 제품만 서버
카탈로그에 반영한다. 전기, 배수, 화학 성분 등 위험 요소가 있는 제품은 강화 기준도
통과해야 한다.
